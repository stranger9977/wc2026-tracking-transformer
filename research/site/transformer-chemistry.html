Transformer Chemistry

The JOI / JDI metrics on the other tabs use event-level VAEP — one number per on-ball action. That throws away ~95% of the game: every off-ball run, every defensive shape adjustment, every moment where a player pulls a defender out of position without touching the ball.

This tab uses a tracking transformer — a model that sees all 22 players + ball at 5 Hz and emits a single expected-threat (xT) prediction per frame plus an attention map showing which players the model is reasoning about. We use that for two things:

  1. Augmented VAEP — fold transformer features into the event-level P-score / P-concede classifiers and measure the AUC lift.
  2. Attention chemistry networks — sum the transformer's pair attention across the tournament. The result is a chemistry metric that captures off-ball signal that JOI/JDI can't see.

These edges are derived frame-by-frame across every match the team played, so they capture pairs that the model "watched together" — including in moments with no on-ball event.

How the transformer fits in

The backbone is a 6-layer transformer encoder over a 23-token state (22 players + 1 ball), each token a 7-feature vector. We trained it to regress max-xT-next-1s as a continuous label (val Spearman ρ = 0.714, +0.098 over an xT-lookup baseline). On this tab we use that model as both a feature source and as an attention extractor — the architecture and training script are at src/wc2026_tracking_transformer/.
